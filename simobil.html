Design basics grid, colors, fonts used in Si.mobil design

The Grid

The grid used in Si.mobil design follows the same principles as the default Bootrstrap grid including the responsive aspect.

The dimensions of the grid and the breakpoints are however custom. The Si.mobil grid system provided utilizes 12 columns that render out at widths of 912px. Below 767px viewports, the columns become fluid and in most cases stack vertically.

Si.mobil specific grid is wrapped in a <div class="container"/> element. To use the default Bootstrap grid behaviour, wrap the row elements in <section /> element.

Fonts

Si.mobil design utilizes custom font SimpleWeb with a base size of 13px and bae line height 18px. The font family is defined in simobil/variables.less file:

@simobilFont "SimbleWeb", Arial, sans-serif
@baseFontSize 13px
@baseLineHeight 18px
Since this documentation is based on Si.mobil specific stylesheet, the samples for the different typography elements can be found in Bootstrap documentation here.

Layout elements Backgrounds, containers, headers...

The whole layout is divided in page blocks, defined by div.page-block. This element enables containing elements to span through the whole width of the browser window if needed. The samples for top bar, main navigation, footer... are wrapped in this element.

Top bar

Top bar is made of two elements spanning through the whole width of the page. The first element .login-bar is hidden by default and opens up when the user clicks on the login (li.user) link. The second element is the div.top-bar containing the site profile switcher .nav-profile (residential/business), login link li.user and the cart link li.cart. with the cart contents dropdown

This JavasSript snippet is used for the top-bar login toggle: 

/** Login toggle **/
$('.top-bar li.user a').click(function(event) {
 event.preventDefault();
 $('.login-bar').slideToggle(300, function() {
  $('.cart')[0].className = $('.cart')[0].className; // IE7 force redraw
 });
});
The cart contents dropdown is toggled by Bootstrap Dropdown plugin. The documentation can be found here.

Floater

The floater element div#selection spans through the whole .container element. It can include an optional div.message element (the orange message with an arrow).

The .scroll element contains the content of the floater and is defined by the basic grid.

When te user scrolls past the starting point of the floater element, the floater sticks to the top of the wiewport and scrolls with the window. This is achieved with a javascript snipped that adds a class .fixed to the div.selection element.

On mobile devices the div.scroll element is hidden, and the floater height is reduced to it minimum height, containing only the .message and the .tools element.

The Javascript:

/** Product list selection floater **/
var $win = $(window)
  , $nav = $('#selection .selection')
  , $navParent = $('#selection')
  , navTop = $('#selection').length && $('#selection').offset().top
  , isFixed = 0;

$win.on('scroll', processScroll);

function processScroll() {
 var scrollTop = $win.scrollTop();
 var toolsHeight = $('.selection .tools').height();
 var toolsTop = navTop - toolsHeight + $('.selection').height();
 if ($('.selection .tools .scroll-top').css('display') == 'none') {
  if (scrollTop >= navTop && !isFixed) {
   isFixed = 1;
   $navParent.css('height', $navParent.height());
   $nav.addClass('fixed');
  } else if (scrollTop <= navTop && isFixed) {
   isFixed = 0;
   $nav.removeClass('fixed');
   $navParent.css('height', 'auto');
  }
 } else {
  if (scrollTop > toolsTop && !isFixed) {
   isFixed = 1;
   $navParent.css('height', $navParent.height());
   $('.selection .tools').addClass('fixed');
  } else if (scrollTop <= (toolsTop+toolsHeight) && isFixed) {
   isFixed = 0;
   $('.selection .tools').removeClass('fixed');
   $navParent.css('height', 'auto');
  }
 }
}

The calendar element

The calendar element is a normal form text input element wrapped in a div element with the class names .field and .d. The calendar component is based on jQueryTools Dateinput plugin.

The markup: 

<form>

 </div>
</form>

Lightbox

Lightbox elements are based on jQueryTools plugin Overlay.

To create a lighbox, you need a trigger element (an element which opens the lightbox). This can be any html element with a class .lightbox-trigger and a rel attribute which contains the id of the lightbox window element.

The lightbox element can be any div element with a class="lightbox" and a specific id. This element must contain a div element with a class="lightbox-content".

The script required: 

<script>
$('.lightbox-trigger[rel]').overlay({
 mask: {
  color: '#333',
  loadSpeed: 200,
  opacity: 0.9
 },
 closeOnClick: false,
 fixed: false
});
</script>

Gallery

Generic gallery widget

Gallery elements are based on jQueryTools plugin Scrollable.
To create a gallery, you need to define 3 elements:

  • a wrapping element with class .scrollable, a .
  • an element with class .items which holds the gallery slides
  • an optional div element with class .navi for gallery navigation icons - circles

The script required: 

<script>
  $('.scrollable')
    .scrollable()
    .navigator('.navi');
</script>

  • Large gallery
